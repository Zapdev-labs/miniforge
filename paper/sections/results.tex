\label{fig:throughput}

All configurations exceed the 10 tok/s target for interactive applications. The variation by prompt type reflects different output patterns: code generation tends toward shorter, more deterministic sequences, while creative writing produces longer, more varied outputs that exercise different computational paths through the model.

Notably, the short code generation scenario achieves 22.5 tok/s mean throughput, more than double the 10 tok/s target. This performance headroom enables responsive interactive applications even under suboptimal conditions (thermal throttling, background processes, etc.).

\textbf{Analysis:} The throughput variation of approximately 19\% between short code and long creative prompts suggests that prompt complexity affects not just the generated output but the generation process itself. Longer contexts require more attention computation, and creative prompts may activate different model pathways than structured code generation. The low standard deviation (4-6\%) indicates consistent performance, essential for predictable user experience.

\begin{table}[h]

\midrule

Working Memory & 1.85 \\
Overhead & 0.42 \\
\midrule
\textbf{Total} & \textbf{6.31} \\
\bottomrule
\end{tabular}
\end{table}

Total memory consumption of 6.31 GB represents only 26\% of the 24GB available budget, leaving over 17GB of headroom. This substantial margin accommodates:
\begin{itemize}
    \item Operating system and background processes
    \item Python interpreter and library overhead
    \item Application working memory for complex tasks
    \item Safety buffers for memory fragmentation
    \item Potential future model loading or concurrent operations
\end{itemize}

\subsubsection{Memory Scaling}

Memory scales linearly with context size at approximately 14 KB/token with turbo3 compression. This linear relationship enables predictable capacity planning:

\begin{itemize}
    \item 32K context: $\approx$0.45 GB KV cache
    \item 64K context: $\approx$0.90 GB KV cache
    \item 128K context: $\approx$1.79 GB KV cache
    \item 192K context: $\approx$2.69 GB KV cache (measured)
\end{itemize}

The linear scaling confirms theoretical predictions and enables reliable context window calculations. At 14KB/token, the full 192K context window requires 2.69 GB, exactly matching our measurements.
